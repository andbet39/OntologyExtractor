# Errori

Tutti gli errori specifici estendono `OntologyExtractorError`, che imposta `name` al nome della classe concreta.

## `ConfigurationError`

Errore di configurazione sollevato da `resolveConfig` o dal costruttore di `OntologyExtractor`. Indica opzioni invalide prima dell'avvio della pipeline.

## `SchemaValidationError`

Errore di validazione dati. Puo nascere da documenti in input invalidi, risposte LLM con shape errata, candidati incoerenti o verdetti di canonicalizzazione non validi.

Esempi:

- documento con `id` vuoto;
- `RelType` senza `domain` o `range`;
- `primaryKeyAttribute` che non riferisce un attributo dichiarato;
- verdict LLM con `targetId` non incluso nei vicini.

## `LlmAdapterError`

Errore legato al provider LLM. La libreria lo usa quando:

- l'estrazione fallisce per tutti i chunk;
- una chiamata di canonicalizzazione LLM fallisce.

Spesso include `cause` con l'errore originale.

## `EmbeddingAdapterError`

Errore legato a embedding o calcolo similarita. Copre fallimenti provider, dimensioni errate, valori non finiti e vettori zero.

## `StoreConflictError`

Errore di conflitto nello store. Esempi:

- `extract` su store non vuoto;
- append di documento duplicato;
- candidati con provenienza non coerente con il documento appendato.

## `ConcurrentJobError`

Errore sincrono sollevato quando si prova ad avviare un secondo job sulla stessa istanza di `OntologyExtractor`.

## `CancelledError`

Errore terminale prodotto da `ExtractionJob.cancel()`. Indica cancellazione cooperativa locale.
