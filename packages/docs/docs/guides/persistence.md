# Persistenza e raffinazione

Il `CandidateStore` e il punto in cui la libreria separa l'estrazione costosa dalla consolidazione. Salvando candidati, puoi ricalcolare l'ontologia senza ripetere tutte le chiamate di estrazione.

## Store in-memory

`InMemoryCandidateStore` e utile per test, CLI locali e sessioni brevi. Mantiene:

- un set di documenti processati;
- una lista di candidati per documento;
- copie difensive in lettura e scrittura.

Lo stato viene perso quando il processo termina.

## Contratti importanti

- `appendCandidates` deve essere atomico a livello documento.
- Un documento gia presente deve essere rifiutato o gestito come conflitto, non duplicato.
- I candidati devono avere `documentId` coerente con il documento appendato.
- `getAllCandidates` deve restituire uno snapshot scollegato dallo stato interno.
- `getDocumentIds` deve includere anche documenti senza candidati.

Queste regole sono cio che rende `refine` sicuro: la pipeline puo saltare documenti gia visti e ricostruire supporto/provenienza senza doppio conteggio.
