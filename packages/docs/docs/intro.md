# Ontology Extractor

`ontology-extractor` e una libreria TypeScript provider-agnostic per estrarre una bozza di TBox da corpus documentali. Non importa direttamente SDK di provider LLM: l'applicazione fornisce un adapter per risposte strutturate e un adapter per embedding, mentre la libreria coordina chunking, estrazione, persistenza dei candidati, canonicalizzazione e pubblicazione dell'ontologia.

## Cosa produce

La libreria emette un oggetto `Ontology` composto da `CanonicalType[]`. Ogni tipo canonico rappresenta un concetto (`NodeType`) o una relazione (`RelType`) con:

- etichetta, definizione e alias;
- attributi semplici o metadati attributo completi;
- categoria, chiave primaria, attributo di label e metadati visuali quando supportati dal testo;
- cardinalita delle relazioni quando inferita dal documento;
- conteggio di supporto e provenienza documentale.

La bozza puo poi essere convertita nel modello interoperabile `OntologyImportModel` tramite `toOntologyImportModel`.

## Quando usarla

Usala quando devi trasformare procedure, SOP, documentazione tecnica, knowledge base o altri testi aziendali in uno schema iniziale da revisionare. La libreria non promette una ontologia finale pronta per produzione: produce un layer strutturato e tracciabile che resta verificabile grazie a `supportCount` e `provenance`.

## Entrypoint pubblici

- `ontology-extractor`: API core, tipi, store in-memory, helper e serializer.
- `ontology-extractor/langchain`: bridge opzionale per modelli LangChain.

## Requisiti runtime

- Node.js `>=20`.
- TypeScript/ESM.
- Un adapter LLM capace di restituire JSON validato da schema.
- Un adapter embedding che dichiara `dimensions` e restituisce vettori numerici finiti.

Per ragioni di sicurezza e controllo dati, la libreria core non legge variabili ambiente, non apre connessioni di rete e non conserva segreti. Queste responsabilita restano nell'applicazione o negli adapter.
